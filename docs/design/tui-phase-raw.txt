A full config TUI covering profiles, capabilities, permissions, and external CLI setup is a
    meaningfully larger surface — but it maps perfectly to what ratatui does well:           
                                                                                                       
    ┌──────────────────────────────────────────────┬────────────────────────────────────────┐        
    │                    Screen                    │               Complexity               │          
    ├──────────────────────────────────────────────┼────────────────────────────────────────┤
    │ Profile picker (activate/deactivate)         │ Low — list + toggle, same as gws       │          
    ├──────────────────────────────────────────────┼────────────────────────────────────────┤          
    │ Capability browser (namespace tree drill-in) │ Medium — hierarchical list             │
    ├──────────────────────────────────────────────┼────────────────────────────────────────┤          
    │ Pack install / manage                        │ Medium — list + actions                │          
    ├──────────────────────────────────────────────┼────────────────────────────────────────┤          
    │ External CLI setup (wizard-style)            │ Medium-High — multi-step form          │
    ├──────────────────────────────────────────────┼────────────────────────────────────────┤          
    │ Permissions / policy editor                  │ High — structured form with validation │        
    └──────────────────────────────────────────────┴────────────────────────────────────────┘          
                                                                    
    Lift estimate                                                                                      
                                                                                         
    - ratatui + crossterm add ~2 lightweight deps, no async required                                   
    - A focused v1 (profile manager + capability browser + pack install) is ~600-900 lines — a week of
    solid work                                                                                         
    - The full wizard-style onboarding for external CLIs is another ~400-600 lines on top              
    - The hardest part isn't the widgets — it's the state machine for multi-screen navigation and form
    validation                                                                                         
                                                                                         
    The good news                                                
                            
    The clix data model is already perfect for a TUI:
    - Registry → browsable tree (namespace → capability → detail)
    - Profiles → toggle list
    - Receipts → scrollable log viewer
    - Pack install → progress + confirmation
                                                                                           
    Recommendation                                                                                     
                                                                                                     
    Build it after the dynamic CLI is done — the TUI becomes the graphical face of the same
    registry/profile data. The clix tui command launches it; everything else still works headlessly for
     agents. The two modes complement each other perfectly: TUI for humans setting things up, MCP/CLI
    for agents running things.                

    Not high complexity for what you'd get — ratatui is well-documented and the clix data layer is
    already clean enough to drive it directly.
